## 新增需求

### 需求：左侧输入区域固定布局
左侧输入区域必须固定在页面左侧，使用固定窗口、全屏高度，宽度必须自适应剩余空间。

#### 场景：左侧输入区域占据屏幕左侧
- **当** 用户打开应用
- **那么** 左侧输入区域固定显示在页面左侧
- **并且** 输入区域高度为全屏高度（100vh）
- **并且** 输入区域宽度自适应（填充可用空间减去预览区域固定宽度）

#### 场景：左侧输入区域不随页面滚动
- **当** 页面内容超出视口高度
- **那么** 左侧输入区域保持固定位置
- **并且** 输入区域不随页面滚动

### 需求：右侧预览区域自适应高度
右侧预览区域必须使用固定宽度，高度必须自动适应内容，确保所有文字内容完整显示不被裁切。

#### 场景：预览区域高度自适应内容
- **当** 用户输入的 markdown 内容长度增加
- **那么** 预览区域高度自动增长
- **并且** 所有文字内容完整显示在预览区域内
- **并且** 不会出现内容被裁切或需要滚动查看的情况

#### 场景：预览区域固定宽度
- **当** 用户打开应用
- **那么** 右侧预览区域使用固定宽度（与当前实现一致）
- **并且** 预览区域位于页面右侧

### 需求：下载按钮浮动显示
"Download as Image" 按钮必须浮动显示在页面右上角。

#### 场景：下载按钮位置
- **当** 用户打开应用
- **那么** "Download as Image" 按钮固定在页面右上角
- **并且** 按钮为浮动按钮（不占用布局空间）
- **并且** 按钮始终可见，不随页面滚动而移动

#### 场景：下载按钮功能保持不变
- **当** 用户点击浮动下载按钮
- **那么** 系统生成当前预览区域的 PNG 图片
- **并且** 自动下载图片文件

### 需求：代码注释规范
所有关键代码段必须包含清晰的中文注释，解释代码的功能和用途。

#### 场景：组件结构注释
- **当** 开发者阅读源代码
- **那么** 每个主要组件结构必须有中文注释
- **并且** 注释说明该结构的功能和作用

#### 场景：关键逻辑注释
- **当** 代码包含业务逻辑或状态管理
- **那么** 关键逻辑必须有中文注释
- **并且** 注释解释逻辑的目的和实现方式

#### 场景：样式类注释
- **当** 使用复杂的 Tailwind CSS 类组合
- **那么** 必须添加中文注释说明样式的作用

## 修改需求

### 需求：整体布局响应式设计
应用必须采用左右分栏布局，左侧固定、右侧内容自适应。

#### 场景：左右分栏布局
- **当** 用户打开应用
- **那么** 页面分为左右两个区域
- **并且** 左侧为输入区域（固定位置、全屏高度、宽度自适应）
- **并且** 右侧为预览区域（固定宽度、高度自适应内容）
