## 为什么

当前 Markdown to Image 应用的布局存在几个问题：
1. 右侧图片预览区域使用固定高度，当内容较长时会被裁切，用户无法看到完整的预览效果
2. 左侧输入框使用固定宽度（`w-112.5`），在不同屏幕尺寸下体验不佳
3. "Download as Image" 按钮位置固定在输入框下方，不够醒目且占用额外空间
4. 代码缺少注释，维护性较差

## 变更内容

### 新增功能
无

### 修改
- **左侧输入区域布局**：改为固定位置、固定窗口、全屏高度，宽度自适应
- **右侧预览区域布局**：改为自适应高度，确保所有文字内容完整显示不被裁切，使用固定尺寸宽度
- **"Download as Image" 按钮**：移动到右上角作为浮动按钮
- **代码注释**：为所有关键代码段添加清晰的中文注释

## 功能 (Capabilities)

### 新增功能
无

### 修改功能
- `markdown-preview-layout`: 修改 markdown 预览布局功能的需求规范
  - 将预览区域从固定高度改为自适应高度
  - 调整整体布局结构，支持左侧固定、右侧自适应的响应式布局
  - 重新定位下载按钮为浮动按钮

## 影响

### 受影响的代码
- `src/App.tsx`: 主要修改文件，涉及布局结构和组件实现

### 依赖项
- `react-markdown`: 用于 markdown 渲染（无变更）
- `html-to-image`: 用于图片导出（无变更）
- `tailwindcss`: 用于样式实现（可能需要自定义样式类）

### 用户体验
- 改善长文档的预览体验，内容不再被裁切
- 提供更直观的下载按钮位置
- 更好的响应式布局支持
