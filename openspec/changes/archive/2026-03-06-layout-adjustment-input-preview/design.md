## 上下文

### 当前状态
当前 Markdown to Image 应用使用 Tailwind CSS 实现布局：
- 左侧输入区域：使用绝对定位（`absolute top-32 left-8`）、固定宽度（`w-112.5`）、固定高度（`h-96`）
- 右侧预览区域：使用左边距（`ml-120`）、固定宽度（`w-250`）、高度未明确设置（可能被内容撑开但有容器限制）
- 下载按钮：位于输入框下方，占用布局空间

### 约束条件
- 必须保持现有功能不变（markdown 渲染、图片导出）
- 使用 Tailwind CSS 进行样式实现
- 单文件应用（`src/App.tsx`）
- 不引入新的外部依赖

### 技术栈
- React 18+
- React Markdown（markdown 渲染）
- html-to-image（图片导出）
- Tailwind CSS（样式）

## 决策

### 1. 布局策略：Flexbox vs Grid

**决策：使用 Flexbox 布局**

**理由：**
- 当前实现已使用 Tailwind CSS，Flexbox 与其完美集成
- 布局需求简单（左右两栏），Flexbox 足够应对
- 代码改动最小，易于维护

**替代方案：** CSS Grid
- 优点：更适合复杂二维布局
- 缺点：对于简单两栏布局过度设计，代码更复杂

### 2. 左侧输入区域实现方案

**决策：使用 `fixed` 定位 + `flex` 布局**

**实现细节：**
```tsx
// 左侧容器：固定定位、全屏高度
<div className="fixed left-0 top-0 h-screen w-1/2 p-8">
  <textarea className="w-full h-full" />
</div>
```

**理由：**
- `fixed` 定位确保输入区域不随页面滚动
- `w-1/2` 实现宽度自适应（占据 50% 可用空间）
- `h-screen` 确保全屏高度
- 使用 `flex` 布局使 textarea 填充容器

### 3. 右侧预览区域实现方案

**决策：移除高度限制 + 固定宽度**

**实现细节：**
```tsx
// 右侧容器：固定宽度、高度自适应
<div className="ml-1/2 w-[1000px] min-h-screen">
  <div ref={previewRef} className="w-[1000px]">
    {/* 预览内容 */}
  </div>
</div>
```

**理由：**
- 移除 `max-h-xxx` 等高度限制类
- 保持固定宽度（`w-[1000px]`，约等于 `w-250`）
- 使用 `min-h-screen` 确保至少占据全屏高度
- 内容自然撑开容器高度

### 4. 浮动按钮实现方案

**决策：使用 `fixed` 定位 + `z-index`**

**实现细节：**
```tsx
// 浮动按钮：右上角固定定位
<button className="fixed top-8 right-8 z-50">
  Download as Image
</button>
```

**理由：**
- `fixed` 定位使按钮始终在视口右上角
- `z-50` 确保按钮在最上层
- 不占用布局空间，不影响其他元素

### 5. 代码注释规范

**决策：使用 JSDoc 风格的中文注释**

**实现细节：**
```tsx
// 状态：存储用户输入的 markdown 文本
const [markdown, setMarkdown] = useState('# Hello World');

// 引用：预览区域 DOM 元素，用于截图导出
const previewRef = useRef<HTMLDivElement>(null);
```

**理由：**
- JSDoc 风格注释易于阅读和维护
- 中文注释确保团队理解一致
- 在关键逻辑处添加解释性注释

## 风险 / 权衡

### 风险 1：小屏幕设备显示问题
**风险：** 在小屏幕设备上，左右分栏布局可能过于拥挤

**缓解措施：**
- 当前设计主要针对桌面端
- 未来可考虑添加响应式断点（`md:`、`lg:` 前缀）
- 本变更不包含移动端适配

### 风险 2：超长内容性能问题
**风险：** 预览区域高度无限制，超长内容可能影响渲染性能

**缓解措施：**
- React Markdown 本身性能良好
- 未来可考虑虚拟滚动（如需要）
- 本变更不处理性能优化

### 风险 3：固定宽度预览区域可能在不同 DPI 下显示不一致
**风险：** 使用像素值（`1000px`）可能导致高 DPI 屏幕显示问题

**缓解措施：**
- 当前实现已使用像素值，保持一致性
- 未来可考虑使用 `rem` 或 `em` 单位
- 本变更保持现有单位系统
